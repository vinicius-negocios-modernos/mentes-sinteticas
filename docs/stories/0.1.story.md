## Descricao
Como desenvolvedor do Mentes Sinteticas, quero restaurar o sistema a um estado funcional re-ingerindo os File URIs expirados do Gemini, para que o chat com as mentes sinteticas volte a funcionar apos mais de 2 meses inoperante.

## Contexto
O sistema esta 100% inoperante desde 2026-01-02. Os File URIs do Gemini expiram 48 horas apos upload. O campo `last_updated` no `minds_manifest.json` e `2025-12-31T03:58:38.971Z`, o que significa que todos os 21 arquivos da knowledge base expiraram ha mais de 2 meses. Nenhuma conversa com nenhuma mente funciona.

Esta e a story mais urgente de todo o epic -- sem ela, nenhum trabalho subsequente pode ser validado.

**ALERTA:** O script `ingest_mind.ts` pode nao ser idempotente. E necessario validar antes de re-executar para evitar duplicatas no Gemini File API.

## Notas Tecnicas
- O script de ingestion esta em `scripts/ingest_mind.ts`. Ele le `data/minds_manifest.json` e faz upload de arquivos da `knowledge_base/` para o Gemini File API.
- Os File URIs do Gemini tem TTL de 48 horas. Apos este periodo, eles retornam erro ao serem referenciados.
- O manifest usa `localPath` com caracteres especiais (acentos, dois-pontos). Cuidado com encoding.
- **IMPORTANTE:** Parar o servidor de desenvolvimento ANTES de rodar o script de ingestion para evitar race condition na leitura/escrita do manifest JSON.
- O footer com modelo errado esta em `src/app/page.tsx:69` (hardcoded "Gemini 1.5 Pro"). O modelo real e `gemini-2.0-flash` (definido em `src/lib/gemini.ts:55`).
- Os console.logs a remover estao em `src/app/actions.ts` (debug logging) e `src/components/ChatInterface.tsx`.
