## Descricao
Como desenvolvedor do Mentes Sinteticas, quero corrigir problemas rapidos de qualidade de codigo, tipagem, e navegacao, para que o projeto tenha uma base limpa antes de iniciar trabalhos estruturais maiores.

## Contexto
O prototipo tem varios problemas de baixo esforco e alto impacto que podem ser resolvidos rapidamente. Tipos `any` espalhados pelo codebase prejudicam a manutenibilidade. A navegacao usa `<a>` tags em vez de `next/link`, causando full page reload com flash branco contra o fundo escuro (#030014). O script de ingestion captura `last_updated` mas nao captura `expirationTime` por arquivo, tornando impossivel saber quando cada URI vai expirar. Reads sincronos (`readFileSync`) bloqueiam o event loop, o que e critico se deployado em serverless (Vercel).

## Debitos Resolvidos
- **SYS-018**: `any` types throughout codebase -- `history: any[]`, `error: any`, manifest data typed as `any` (MEDIO, 3-4h)
- **UX-007**: No `next/link` usage -- navegacao via `<a>` tags causa full page reload (ALTO, 1-2h)
- **DB-001**: Manifest JSON nao tem `expires_at` por arquivo -- impossivel saber qual URI expirou (CRITICO, 2-3h)
- **SYS-014**: Synchronous filesystem reads -- `readFileSync` em `gemini.ts:33,39` bloqueia event loop (MEDIO, 1-2h)

## Notas Tecnicas
- **Tipos do Gemini API:** A resposta de `files.create()` retorna um objeto `File` que inclui `expirationTime` como string ISO 8601. Ref: `@google/generative-ai` SDK types.
- **Manifest structure atual:** `minds_manifest.json` tem estrutura `{ minds: [{ displayName, systemInstruction, files: [{ fileUri, mimeType, localPath }], last_updated }] }`. Adicionar `expires_at` dentro de cada file entry.
- **readFileSync locations:** `src/lib/gemini.ts` linhas ~33 e ~39. Usados para ler o manifest e potencialmente arquivos de knowledge base.
- **next/link:** Importar de `next/link`. Substituir `<a href="/chat/...">` por `<Link href="/chat/...">`. Atencao: `next/link` prefetches por default em producao.
- **Cuidado com imports:** O arquivo `gemini.ts` e usado em server actions. Confirmar que async reads funcionam corretamente no contexto de Server Components / Server Actions do Next.js App Router.
