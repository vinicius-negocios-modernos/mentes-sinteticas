## Descricao
Como usuario do Mentes Sinteticas, quero que o sistema lide com erros de forma graceful e me apresente mensagens uteis, para que eu saiba o que esta acontecendo quando algo da errado em vez de ver telas brancas ou mensagens genericas.

## Contexto
O prototipo nao tem nenhuma estrategia de error handling. O servidor crasha inteiro se a variavel `GEMINI_API_KEY` estiver ausente (throw em module-level em `gemini.ts`). Nao existem React Error Boundaries, nem `error.tsx`, nem `not-found.tsx`, nem `loading.tsx`. Quando a API do Gemini falha, o client mostra "Falha na conexao neural." hardcoded independente do tipo de erro. O error message do script de ingestion diz "Video processing failed." (copy-paste do exemplo de video do Google).

Estes problemas sao de baixo esforco e alta visibilidade -- resolver antes de trabalhos maiores evita debugging frustrante.

## Debitos Resolvidos
- **SYS-007**: Module-level crash on missing env var -- `gemini.ts` throws at import se `GEMINI_API_KEY` ausente (ALTO, 2h)

## Notas Tecnicas
- **Lazy initialization em gemini.ts:** Em vez de `const genAI = new GoogleGenerativeAI(apiKey)` no top-level, usar uma funcao `getGeminiClient()` que cria a instancia sob demanda e lanca erro descritivo se a key estiver ausente.
- **Error Boundary pattern do Next.js App Router:** `error.tsx` deve exportar um Client Component (`'use client'`) que recebe `error` e `reset` como props. Ref: Next.js docs.
- **not-found.tsx:** Ativado automaticamente pelo Next.js quando `notFound()` e chamado ou quando uma rota nao existe.
- **Classificacao de erros:** O Gemini SDK pode lancar varios tipos de erro. Verificar `error.message` para patterns como "API key not valid", "RESOURCE_EXHAUSTED" (rate limit), "NOT_FOUND" (mind/file URI invalido).
- **"Video processing failed"** esta em `scripts/ingest_mind.ts` -- e um artefato de copy-paste do exemplo de video processing do Google AI docs.
