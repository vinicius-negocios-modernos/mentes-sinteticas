# Story 2.1: Setup Supabase + Schema Inicial + Drizzle ORM
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 2 -- Fundacao (DB, Auth, CI/CD)
**Status:** Done

**Criado por:** @sm (River)

## Descricao
Como desenvolvedor do Mentes Sinteticas, quero configurar o Supabase como banco de dados PostgreSQL com Drizzle ORM e criar o schema inicial com as 5 tabelas fundamentais, para que o sistema tenha uma fundacao de dados robusta substituindo o modelo file-based atual.

## Contexto
O sistema atualmente opera sem banco de dados -- tudo e file-based via `data/minds_manifest.json`. Isso causa SPOF (single point of failure), impossibilidade de concorrencia, e impede features como historico de conversas, autenticacao e multi-tenancy. Esta story e a FUNDACAO da Fase 2: todas as outras stories de DB, Auth e CI/CD dependem dela.

Stack consensual definida no Brownfield Discovery: Supabase (PostgreSQL) + Drizzle ORM. O projeto usa Next.js 16.1.1, TypeScript 5, Zod (ja configurado).

## Debitos Resolvidos
- **CROSS-002** (parcial): Ausencia de persistencia estruturada -- setup inicial do banco
- **DB-003**: SPOF no minds_manifest.json -- dados migrados para PostgreSQL
- **DB-005**: Sem concorrencia possivel -- PostgreSQL resolve nativamente
- **DATABASE SETUP**: Nova arquitetura de dados com schema relacional

## Notas Tecnicas
- O `minds_manifest.json` contem 1 mente ("Antonio Napole") com 21 knowledge documents. O seed deve mapear os campos corretamente.
- `personality_traits` no manifest e um array de strings -- armazenar como `jsonb` no PostgreSQL.
- O campo `system_prompt` pode ser longo (>1000 chars) -- usar `text`, nao `varchar`.
- `file_uri_cache` substitui os campos `fileUri`/`mimeType`/`expiresAt` que hoje vivem dentro do manifest JSON.
- RLS requer que `conversations.user_id` e `messages` (via join) referenciem `auth.uid()`. Para esta story, auth ainda nao esta implementada -- as policies devem existir mas serao efetivas somente apos Story 2.2 (Auth).
- Drizzle config deve apontar para `DATABASE_URL` do `.env.local` (connection pooler do Supabase, porta 6543).
- Usar `pgTable` do `drizzle-orm/pg-core` para definicao dos schemas.
