# Story 2.4: File URI Auto-Renewal (Edge Function Cron)
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 2 -- Fundacao (DB, Auth, CI/CD)
**Status:** InProgress
**Estimativa:** 4 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como sistema Mentes Sinteticas, quero que os File URIs do Gemini sejam renovados automaticamente antes de expirar para que o chatbot nunca fique indisponivel por URIs expirados, eliminando a necessidade de re-ingestao manual a cada 48h.

## Contexto
Os File URIs do Gemini Files API expiram a cada 48h (debito SYS-010 CRITICO). Na Story 0.1 foi feita re-ingestao manual dos 21 documentos de conhecimento. A tabela `file_uri_cache` (Story 2.1) ja existe com campos: `id`, `knowledge_document_id`, `file_uri`, `mime_type`, `expires_at`, `created_at`, `updated_at`. Porem nao existe automacao -- cada 48h os URIs expiram e o sistema quebra silenciosamente. Esta story implementa uma Edge Function com cron que renova URIs automaticamente, alem de migrar `gemini.ts` para ler URIs do DB ao inves do `minds_manifest.json`.

## Debitos Resolvidos
- **SYS-010** (CRITICO): Gemini File URIs expire in 48h -- fix permanente via cron
- **DB-006**: No automated refresh -- Edge Function com schedule

## Notas Tecnicas
- Schema `file_uri_cache`: `id` (uuid PK), `knowledge_document_id` (FK), `file_uri` (text), `mime_type` (text), `expires_at` (timestamptz), `created_at`, `updated_at`.
- Arquivos fonte em `knowledge_base/` -- PDFs e textos do Antonio Napole (21 docs).
- Gemini Files API: `uploadFile(path)` retorna `{ uri, mimeType, expirationTime }`.
- Buffer de 6h: query `WHERE expires_at < NOW() + INTERVAL '6 hours'`.
- Cron 12h garante 2 checks por ciclo de 48h -- margem confortavel.
- Edge Function precisa de `GEMINI_API_KEY` como secret no Supabase.
- Trigger manual: `supabase functions invoke refresh-file-uris` ou HTTP POST.
- Fallback em `gemini.ts`: se DB indisponivel, tentar `minds_manifest.json` como backup.
- Monitoramento: campo `consecutive_failures` na tabela ou log estruturado.
- Supabase Edge Functions usam Deno runtime -- imports via URL ou npm specifiers.
