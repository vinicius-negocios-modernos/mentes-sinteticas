# Story 2.5: Knowledge Base Backup + Migracao Manifest para DB
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 2 -- Fundacao (DB, Auth, CI/CD)
**Status:** InProgress
**Estimativa:** 3 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como sistema Mentes Sinteticas, quero que os documentos de conhecimento originais (PDFs/texto) sejam armazenados no Supabase Storage e que toda leitura de dados de mentes venha do DB (nao mais do `minds_manifest.json`) para eliminar o single point of failure do arquivo JSON local e garantir backup seguro dos documentos fonte.

## Contexto
Atualmente todos os dados de mentes vivem em `data/minds_manifest.json` (SPOF -- debito DB-003). Os documentos de conhecimento estao apenas em `knowledge_base/` local (sem backup -- debito DB-002, sem versionamento -- debito CROSS-006). A Story 2.1 criou as tabelas `minds`, `knowledge_documents` e `file_uri_cache`. A Story 2.4 migrou `gemini.ts` para ler File URIs do DB com fallback ao manifest. O seed script (`scripts/seed-db.ts`) ja popula as 3 tabelas. Esta story completa a transicao: backup dos docs no Supabase Storage, leitura de mentes pelo DB em todo o codigo, e deprecacao do manifest.

## Debitos Resolvidos
- **CROSS-006**: KB management -- sem admin UI, sem versionamento (backup via Storage)
- **DB-002**: No backup strategy -- docs armazenados no Supabase Storage
- **DB-003**: SPOF -- single JSON file substituido por DB como source of truth
