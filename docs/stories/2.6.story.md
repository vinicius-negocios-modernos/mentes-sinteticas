# Story 2.6: Rate Limiting Persistente + Input Validation com Zod
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 2 -- Fundacao (DB, Auth, CI/CD)
**Status:** Done
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como administrador do Mentes Sinteticas, quero que todas as entradas de usuario sejam validadas com Zod e que exista rate limiting persistente (DB-backed) nas server actions para prevenir abuso da Gemini API, prompt injection e payloads malformados ŌĆö eliminando os debitos SEC criticos SYS-002 e SYS-003.

## Contexto
Atualmente as server actions em `src/app/actions.ts` nao validam inputs (mensagens, mindId, conversationId). Nao existe rate limiting ŌĆö qualquer usuario autenticado pode fazer chamadas ilimitadas a API Gemini, gerando risco de exaustao de creditos. A Story 2.1 criou o DB (Supabase + Drizzle), a Story 2.2 implementou auth com middleware. Esta story adiciona a camada de protecao: validacao de inputs + rate limiting persistente usando o DB ja existente.

## Debitos Resolvidos
- **SYS-002**: No rate limiting ŌĆö sem throttling per-user. Chamadas ilimitadas a API.
- **SYS-003**: No input validation on messages ŌĆö sem limites de tamanho, content filtering, ou sanitizacao.
- **SYS-004**: No input sanitization on mindId URL parameter ŌĆö mindId passado sem validacao.

## Notas Tecnicas
- Zod ja esta no projeto (`package.json`) ŌĆö usado em `src/lib/config.ts` para env validation.
- Server actions (`actions.ts`) sao o unico ponto de entrada da API ŌĆö nao existem API routes separadas (exceto `auth/callback/route.ts`).
- Rate limiting usa sliding window: query `rate_limits` WHERE user_id + action + window_start > (now - window_size). Se count >= limit, rejeitar.
- Cleanup de registros antigos: deletar rows com window_start > 24h no passado (pode ser lazy ŌĆö na proxima verificacao).
- Limites sugeridos: 20 msgs/min (burst protection), 200 msgs/hora (sustained protection). Configuravel via env vars.
- `ErrorType` union type em `src/lib/types.ts` ja inclui `RATE_LIMITED` (criado na Story 1.2).
- Sanitizacao de mensagens: `message.replace(/[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g, '').trim()` ŌĆö preserva newlines (\n, \r).
- Tabela `rate_limits` nao precisa de RLS ŌĆö acessada apenas via server actions (server-side).
- Index composto `(user_id, action, window_start)` otimiza a query principal do rate limiter.
