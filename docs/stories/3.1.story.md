# Story 3.1: Design System (shadcn/ui + Design Tokens + Tailwind Config)
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 3 -- Core UX + Chat Experience
**Status:** Done
**Estimativa:** 8 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como desenvolvedor do Mentes Sinteticas, quero estabelecer um design system coeso baseado em shadcn/ui com design tokens e configuracao Tailwind v4 — para que todos os componentes da aplicacao compartilhem uma identidade visual consistente ("Dark Academia meets Sci-Fi") e o desenvolvimento de novas features na Fase 3+ seja rapido, acessivel e visualmente coerente.

## Contexto
A aplicacao atualmente usa classes Tailwind raw sem design system, sem component library e sem design tokens formalizados. As cores e estilos estao definidos ad-hoc em `globals.css` (`:root` com 7 custom properties) e inline nos componentes. Nao existe theme provider, nao existe dark/light toggle, e os componentes UI (botoes, inputs, cards) sao reimplementados manualmente em cada pagina. Esta story e a PRIMEIRA da Fase 3 — ela estabelece a fundacao visual sobre a qual todas as demais stories (3.2 a 3.7) serao construidas. A Fase 2 (fundacao de dados) esta completa (7/7 stories Done).

## Debitos Resolvidos
- **UI-001**: No design system — componentes UI sem padronizacao
- **UI-002**: No component library — botoes, inputs, cards reimplementados manualmente
- **UI-003**: No design tokens — cores, tipografia, espacamento sem formalizacao
- **UI-004**: No theme infrastructure — sem dark/light toggle, sem theme provider

## Notas Tecnicas
- **Tailwind v4:** Usa CSS-first configuration (sem `tailwind.config.js`). Design tokens devem ser definidos via `@theme` directive ou CSS custom properties em `globals.css`. shadcn/ui v2+ suporta Tailwind v4 nativamente.
- **shadcn/ui + React 19:** shadcn/ui ja suporta React 19. Verificar compatibilidade de `next-themes` com React 19 Server Components (usar `"use client"` no theme provider).
- **Migracao CSS existente:** `globals.css` atual tem 7 custom properties + utilities (`.glass-panel`, `.text-gradient`, scrollbar). Migrar para formato HSL shadcn/ui preservando os valores visuais. Manter `.glass-panel` e `.text-gradient` como utilities customizadas.
- **Fontes:** Projeto ja usa Geist (sans) e Geist_Mono via `next/font/google` em `layout.tsx`. Integrar com design tokens `--font-sans` e `--font-mono`.
- **Paleta "Dark Academia meets Sci-Fi":** Base deep indigo (#030014), roxo (#a855f7) como primary, ciano (#00f0ff) como accent, azul (#3b82f6) como secondary. Cards com glassmorphism (blur + transparencia). Gradiente radial no body.
- **next-themes:** Instalar via `npm install next-themes`. Criar provider client-side (`"use client"`). No layout.tsx, `<ThemeProvider>` deve envolver children mas ficar DENTRO de `<body>`.
- **Sonner:** shadcn/ui usa Sonner como toast library. Adicionar `<Toaster />` no layout.tsx apos instalar.
