# Story 3.2: Extracao de Componentes Reusaveis (Base + Chat)
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 3 -- Core UX + Chat Experience
**Status:** Done
**Estimativa:** 8 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como desenvolvedor do Mentes Sinteticas, quero extrair componentes reusaveis das paginas monoliticas existentes — para que a UI seja modular, consistente e facil de manter, utilizando os componentes shadcn/ui e design tokens estabelecidos na Story 3.1.

## Contexto
Atualmente a aplicacao possui 4 paginas com UI inline: home (`page.tsx`), chat (`chat/[mindId]/page.tsx`), login e signup. Os componentes `ChatInterface` e `ConversationList` ja existem em `src/components/` mas nao utilizam shadcn/ui. A home page tem header, footer, e mind cards inline. A chat page tem header inline com navegacao. Login e signup repetem o mesmo pattern de formulario com inputs/botoes raw (sem shadcn/ui). Esta story extrai componentes reusaveis de todas essas paginas, migrando-os para usar os 8 componentes base shadcn/ui instalados na Story 3.1 (Button, Input, Card, Avatar, ScrollArea, Skeleton, Dialog, Sonner).

## Debitos Resolvidos
- **UI-001**: No design system — componentes UI sem padronizacao (parcial, complementa 3.1)
- **UI-002**: No component library — botoes, inputs, cards reimplementados manualmente
- **MAINT-001**: Paginas monoliticas dificultam manutencao e reutilizacao

## Notas Tecnicas
- **Pure refactor:** Esta story NAO adiciona features novas. Toda funcionalidade existente (chat, login, signup, mind selection, conversation CRUD) deve funcionar identicamente apos o refactor.
- **Estrutura de diretorios:** Componentes organizados por dominio: `layout/` (header, footer), `minds/` (mind-card, mind-tag), `chat/` (chat-message, chat-input, chat-header, chat-interface, conversation-list). Componentes shadcn/ui permanecem em `ui/`.
- **Server vs Client components:** AppHeader, AppFooter, MindCard, MindTag podem ser server components (sem estado). ChatMessage pode ser server component. ChatInput, ChatInterface, ConversationList permanecem client components ("use client") por usarem hooks (useState, useRef, useEffect, useRouter).
- **Migracao de imports:** Atualizar todos os imports nas pages apos mover ChatInterface e ConversationList para novos paths.
- **shadcn/ui usage:** Usar Card (card, card-header, card-title, card-description, card-content) para MindCard. Usar Button para acoes e MindTag. Usar Input para ChatInput e forms. Usar ScrollArea para ChatInterface messages area e ConversationList. Usar Avatar para icones de mente (futuro). Usar Skeleton para loading states (futuro).
- **cn() utility:** Usar `cn()` de `@/lib/utils` para composicao de classes em todos os componentes novos.
- **Login/Signup forms:** Server actions (`signIn`, `signUp`) permanecem inline nas pages como funcoes "use server". Os componentes visuais (Card wrapper, Input fields, Button submit) sao migrados para shadcn/ui.
- **ChatInterface refactor:** A logica de estado (messages, input, isLoading, conversationId, handleSend, scrollToBottom) permanece em ChatInterface. Os sub-componentes (ChatMessage, ChatInput) recebem dados e callbacks via props.
