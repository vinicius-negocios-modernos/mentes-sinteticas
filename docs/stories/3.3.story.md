# Story 3.3: Refactor gemini.ts em Modulos + Streaming com Vercel AI SDK
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 3 -- Core UX + Chat Experience
**Status:** InProgress
**Estimativa:** 13 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario do Mentes Sinteticas, quero ver as respostas das mentes aparecendo em tempo real (token por token) em vez de esperar 5-15 segundos por uma resposta completa — para que a experiencia de conversa seja fluida, responsiva e imersiva. Como desenvolvedor, quero que o modulo monolitico `gemini.ts` (225 linhas) seja refatorado em modulos coesos e testavel — para facilitar manutencao, testes e evolucao futura.

## Contexto
O arquivo `src/lib/gemini.ts` concentra toda a logica de IA em um unico modulo monolitico: inicializacao do cliente Gemini, lookup de minds (DB + manifest fallback), resolucao de File URIs, construcao de system prompt com seguranca, gerenciamento de sessao de chat, e envio de mensagens. O `sendMessage` em `src/app/actions.ts` (Step 7) usa `chat.sendMessage()` que retorna a resposta completa — o usuario fica sem feedback visual durante 5-15s. Esta story refatora o modulo em partes coesas e introduz streaming via Vercel AI SDK, reduzindo o time-to-first-token de 5-15s para < 500ms.

## Debitos Resolvidos
- **ARCH-003**: Monolithic AI module — gemini.ts com multiplas responsabilidades
- **PERF-001**: No streaming — usuario espera resposta completa sem feedback
- **UX-005**: Perceived latency — 5-15s sem indicacao de progresso

## Notas Tecnicas
- **Vercel AI SDK:** O pacote `ai` fornece `streamText`, `StreamingTextResponse`, e o hook `useChat`. O provider `@ai-sdk/google` integra com Google Generative AI. Documentacao: https://sdk.vercel.ai/docs
- **`@ai-sdk/google` vs `@google/generative-ai`:** O Vercel AI SDK usa seu proprio provider (`@ai-sdk/google`) que wrapa a API do Google. Para streaming, usar o provider do Vercel. Para o path nao-streaming legado, pode manter `@google/generative-ai` direto.
- **File URIs no Vercel AI SDK:** O `streamText` do Vercel AI SDK suporta `fileData` parts via o formato de mensagens do provider Google. Verificar compatibilidade da injecao de File URIs no formato esperado pelo `@ai-sdk/google`.
- **Persistencia pos-stream:** Usar o callback `onFinish` do `streamText` para persistir a mensagem completa do assistant no DB apos o stream terminar. A mensagem do user deve ser persistida ANTES de iniciar o stream.
- **Rate limiting na API Route:** A API Route deve replicar os mesmos checks de rate limit de `actions.ts` (Steps 2-3). Extrair a logica de auth + rate limit para um helper reutilizavel se possivel.
- **Backward compatibility:** `src/lib/gemini.ts` deve continuar exportando `createMindChat`, `getMindManifest`, `getAvailableMinds` via re-exports de `@/lib/ai`. Consumidores existentes nao quebram.
- **Security:** A API Route deve validar que o request vem de um usuario autenticado (cookie-based auth via Supabase SSR). Nao expor a rota sem autenticacao.
- **Error handling no stream:** Se o stream falhar mid-way, o client deve exibir erro graceful. O `useChat` do Vercel AI SDK tem `onError` callback.
