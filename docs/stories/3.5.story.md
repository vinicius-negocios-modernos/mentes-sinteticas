# Story 3.5: Chat UX Redesign (Textarea, Avatares, Header, Actions, Scroll)
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 3 -- Core UX + Chat Experience
**Status:** Done
**Estimativa:** 8 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario do Mentes Sinteticas, quero uma experiencia de chat moderna e imersiva — com input multi-linha, avatares distintos para mim e para a mente, um header informativo, acoes contextuais nas mensagens, scroll inteligente e um estado vazio acolhedor — para que conversar com uma mente sintetica seja visualmente rico, intuitivo e alinhado com a estetica Dark Academia meets Sci-Fi do produto.

## Contexto
Os componentes de chat existem em `src/components/chat/` (chat-interface, chat-message, chat-input, chat-header, conversation-list) e sao funcionais: streaming via Vercel AI SDK, error boundaries, toasts Sonner, skeletons de loading. Porem a UI ainda e basica: input e single-line (`Input` do shadcn), mensagens nao tem avatares, o header so mostra nome + botao "Encerrar Sessao", nao ha acoes em mensagens (copy), nao ha botao "scroll to bottom" quando o usuario sobe no historico, e nao ha empty state com sugestoes. O design system (Story 3.1) e os componentes shadcn (Avatar, ScrollArea, Skeleton, Button, Sonner) ja estao disponiveis. O tema dark e o padrao via `next-themes`.

## Debitos Resolvidos
- **UX-006**: Basic chat input — single line, sem multi-line support
- **UX-007**: No visual distinction between user and AI messages beyond color
- **UX-008**: Minimal chat header — no avatar, no description, no status
- **UX-009**: No message actions (copy, timestamp)
- **UX-010**: No scroll-to-bottom affordance
- **UX-011**: No empty chat state / onboarding prompts

## Notas Tecnicas
- **Textarea auto-resize:** Usar `useRef<HTMLTextAreaElement>` + `useEffect` que ajusta `style.height` baseado em `scrollHeight`. Reset height para `auto` antes de medir para permitir shrink. Max height via `max-h-[150px]` + `overflow-y-auto`.
- **Avatar initials:** Helper `getInitials(name: string): string` que retorna as 2 primeiras letras de cada palavra (max 2 chars). Ex: "Antonio Napole" -> "AN".
- **Scroll detection:** Usar `IntersectionObserver` no `messagesEndRef` ou `onScroll` callback no container. `IntersectionObserver` e mais performatico (evita events por pixel).
- **Clipboard API:** `navigator.clipboard.writeText()` retorna Promise. Tratar caso de fallback para browsers sem suporte (try/catch + toast de erro).
- **Timestamps:** O tipo `ChatMessageType` precisara de campo `timestamp?: Date` opcional. Mensagens existentes sem timestamp nao mostram nada. Novas mensagens recebem `new Date()` no momento da criacao.
- **Nao inclui:** Voice mode (Fase futura), tema por mente (Fase futura), sharing de conversa (Fase futura), animacoes Framer Motion avancadas (Fase 4/5).
