# Story 4.7: Loading/Suspense Boundaries + Greetings Personalizados
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 4 -- Qualidade + Mobile + Acessibilidade
**Status:** Done
**Estimativa:** 4 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario do Mentes Sinteticas, quero que a aplicacao exiba estados de carregamento adequados (skeletons/spinners) durante a navegacao entre paginas e ao carregar dados assincronos, e que cada mente sintetica me receba com uma saudacao personalizada refletindo sua personalidade e expertise — para que a experiencia seja fluida, sem telas em branco, e imersiva desde o primeiro momento de interacao.

## Contexto
O projeto atualmente nao possui nenhum arquivo `loading.tsx` para route segments do Next.js App Router, nem Suspense boundaries envolvendo operacoes assincronas. Isso resulta em telas em branco durante navegacao e carregamento de dados, prejudicando a experiencia do usuario.

**Debitos tecnicos resolvidos:**
- **SYS-009**: No loading states (loading.tsx). Sem Suspense boundaries para async server components.
- **UX-009**: No loading/Suspense boundaries (cross-ref SYS-009).

**O que ja existe (dependencias resolvidas):**
- Skeleton components em `src/components/skeletons/`: `chat-messages-skeleton`, `mind-card-skeleton`, `conversation-list-skeleton` (Story 3.4)
- Error boundaries e error pages (Story 3.4)
- Design System com shadcn/ui (Story 3.1)
- Chat UX com streaming (Stories 3.3/3.5)
- Chat empty state component em `src/components/chat/chat-empty-state.tsx` — atualmente exibe saudacao generica igual para todas as mentes
- Schema DB: tabela `minds` ja possui campo `greeting` (text, nullable) — atualmente null para todas as mentes
- Minds carregadas do Supabase via `src/lib/services/minds.ts`

**Escopo desta story:**
1. Criar `loading.tsx` para todos os route segments
2. Adicionar Suspense boundaries em componentes de pagina com data fetching assincrono
3. Implementar greetings personalizados por mente no chat empty state
4. Definir greetings para cada mente no DB seed ou config

## Notas Tecnicas
- **`loading.tsx` no App Router:** Next.js automaticamente usa `loading.tsx` como fallback UI enquanto o conteudo do route segment carrega. Funciona como um Suspense boundary automatico em torno do `page.tsx`. Nao requer nenhuma configuracao adicional — basta criar o arquivo no diretorio do route segment.
- **Suspense com Server Components:** Para que `<Suspense>` funcione com data fetching, o componente async deve ser um Server Component separado. Exemplo: extrair `MindCardsGrid` como async Server Component e envolver com `<Suspense>` na page.
- **Streaming e Suspense:** O `useChat` do Vercel AI SDK e um hook client-side. Suspense boundaries nao interferem com hooks client-side — eles so afetam Server Components async. O streaming continuara funcionando normalmente.
- **Greetings config vs DB:** A tabela `minds` ja tem campo `greeting` (text, nullable). Para esta story, definir greetings em config (TypeScript map) por simplicidade e testabilidade. Migrar para DB e escopo de uma futura story de admin panel.
- **Skeleton components existentes:** `MindCardSkeleton` renderiza 1 card skeleton. Para a home page, renderizar um grid de 3-6 skeletons para simular o layout real. `ChatMessagesSkeleton` e `ConversationListSkeleton` ja renderizam layouts completos.
- **Mind identification:** O chat usa `mindId` no URL (que e o nome URL-encoded, ex: "Antonio%20Napole"). O mapa de greetings pode usar o slug ou nome como chave. Preferir slug para consistencia.
- **Suggested prompts por mente:** As 4 sugestoes atuais sao genericas. Personalizar para cada mente (ex: para Aristoteles, perguntas sobre etica e logica; para Tesla, sobre eletricidade e invencao). Manter fallback com as genericas atuais para mentes sem prompts customizados.
