# Story 5.1: Branding (Favicon, App Icons, Meta Tags, SEO, Open Graph)
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 5 -- Polish & Launch Readiness
**Status:** Done
**Estimativa:** 3 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario ou visitante do Mentes Sinteticas, quero que o site tenha branding visual completo (favicon, app icons, manifest) e meta tags otimizados para SEO e compartilhamento social (Open Graph, Twitter Cards, structured data) — para que o app tenha aparencia profissional em browsers, bookmarks, home screens, resultados de busca e previews de links compartilhados em redes sociais.

## Contexto
O projeto atualmente usa o favicon e meta tags padrao do Next.js, sem branding customizado nem otimizacao de SEO. Isso foi identificado no technical-debt-assessment como item de polish pre-lancamento.

**Problemas atuais:**
- **Favicon generico:** O favicon padrao do Next.js nao representa a identidade visual "Dark Academia meets Sci-Fi" do projeto.
- **Sem app icons:** Nao ha apple-touch-icon nem android icons, resultando em icones genericos quando usuarios adicionam o app a home screen.
- **Sem manifest:** Nao ha web app manifest, impedindo instalacao como PWA e personalizacao de splash screens.
- **Meta tags incompletos:** Faltam meta tags de SEO (description, author), Open Graph, e Twitter Cards. Links compartilhados em redes sociais aparecem sem preview.
- **Sem structured data:** Nao ha JSON-LD schema, reduzindo visibilidade em buscadores.
- **Sem robots.txt/sitemap:** Crawlers nao tem orientacao sobre quais paginas indexar.

**O que ja existe (dependencias resolvidas):**
- Design System com HSL tokens e tema dark default: `src/lib/` + shadcn/ui (Story 3.1)
- Layout principal: `src/app/layout.tsx` com metadata basica do Next.js
- Mobile-first responsive design (Story 4.5)
- next.config.ts existente

**Escopo desta story:**
Criacao de assets visuais (favicon SVG/ICO, app icons), web app manifest, meta tags completos (SEO + OG + Twitter Cards), structured data JSON-LD, robots.txt, sitemap.xml, e configuracao de metadata base URL no next.config.ts.

## Notas Tecnicas
- **Next.js Metadata API:** Next.js 14+ (e 16.x) suporta metadata via export `metadata` ou `generateMetadata` no `layout.tsx`. Favicons, apple-icon, e manifest sao detectados automaticamente quando colocados em `app/` com nomes convencionais (`icon.svg`, `apple-icon.png`, `manifest.ts`). Referencia: Next.js File Conventions.
- **Favicon SVG:** Usar SVG como favicon principal permite adaptacao a dark/light mode via `@media (prefers-color-scheme)` dentro do SVG. O ICO fallback cobre IE e browsers antigos.
- **OG Image:** A imagem Open Graph (1200x630) deve ter o logo/branding do app com fundo escuro (tema Dark Academia). Pode ser criada como SVG e convertida, ou como PNG estatico em `public/og-image.png`. Next.js tambem suporta `opengraph-image.tsx` para geracao dinamica via `ImageResponse`.
- **Manifest theme_color/background_color:** Usar os valores HSL do design system convertidos para hex. `theme_color` afeta a barra de status em mobile; `background_color` aparece no splash screen.
- **robots.ts e sitemap.ts:** Next.js suporta geracao programatica via funcoes exportadas desses arquivos. Isso e preferivel a arquivos estaticos pois permite logica condicional e referencia a env vars.
- **metadataBase:** Essencial para que OG images usem URLs absolutas. Configurar como `new URL(process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000')` no layout.tsx metadata ou no next.config.ts.
- **JSON-LD:** Inserir como `<script type="application/ld+json">` no layout.tsx. Alternativa: usar a propriedade `metadata.other` ou um componente dedicado. Validar com Google Rich Results Test.
- **Assets de icone:** Para gerar ICO a partir de SVG, considerar ferramentas como `sharp`, `svg2png`, ou servicos online. O ICO deve conter pelo menos 16x16 e 32x32. Alternativa: usar apenas SVG favicon (suportado por todos os browsers modernos) e manter ICO minimo.
- **Env var:** `NEXT_PUBLIC_APP_URL` — URL de producao do app (ex: `https://mentes-sinteticas.vercel.app`). Adicionar ao `.env.example`.
- **Lighthouse target:** Apos implementacao, verificar Lighthouse SEO score > 90 e Performance score > 90. Meta tags completos e manifest contribuem diretamente para o score.
- **Paginas autenticadas no sitemap:** Excluir `/chat/*` e `/api/*` do sitemap. Incluir apenas paginas publicas: `/`, `/login`, `/signup`.
