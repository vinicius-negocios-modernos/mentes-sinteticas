# Story 5.2: Empty States + Onboarding / Tutorial
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 5 -- Polish & Launch Readiness
**Status:** Done
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como novo usuario do Mentes Sinteticas, quero ser recebido com empty states visuais que me guiem para a proxima acao, e um onboarding leve na minha primeira visita que explique o conceito das "Mentes Sinteticas" e como usar a plataforma — para que eu entenda rapidamente o valor do produto e consiga iniciar minha primeira conversa sem friccao.

## Contexto
O projeto atualmente nao possui tratamento adequado de empty states nem qualquer onboarding para novos usuarios, identificado como debito tecnico UX-NEW-006 ("No onboarding or tutorial for new users").

**Problemas atuais:**
- **Empty states genericos:** Quando o usuario nao tem conversas, a interface mostra estados vazios sem orientacao visual ou CTAs claros. O `chat-empty-state.tsx` existe mas pode ser aprimorado com visuais alinhados ao tema Dark Academia.
- **Zero onboarding:** Um novo usuario apos login nao recebe nenhuma orientacao sobre o que sao as "Mentes Sinteticas", como escolher uma mente, ou como iniciar uma conversa. A curva de aprendizado depende inteiramente de exploracao.
- **Primeira impressao fraca:** A home page (/) e a lista de conversas sem itens nao comunicam valor nem convidam a acao.
- **Busca sem feedback:** Quando uma busca nao retorna resultados, nao ha empty state adequado.

**O que ja existe (dependencias resolvidas):**
- Design system completo: shadcn/ui com tokens HSL, dark default (Story 3.1)
- Componentes extraidos: `chat-empty-state.tsx`, `conversation-list.tsx`, `mind-card.tsx` (Story 3.2)
- Chat UX redesenhado com textarea, avatars, scroll, empty state basico (Story 3.5)
- Mobile-first responsivo com touch targets 44px, safe areas, drawer (Story 4.5)
- Skeletons existentes: `src/components/skeletons/` (chat-messages, mind-card, conversation-list)
- Auth com Supabase: login/signup pages, middleware protege /chat/* (Story 2.2)

**Debito resolvido:**
- **UX-NEW-006:** No onboarding or tutorial for new users.

**Escopo desta story:**
Empty states com ilustracoes/visuais e CTAs para 4 cenarios (home, conversation list, chat, search), onboarding modal com 3-4 steps para primeiro login, deteccao de first-login, persistencia do estado de onboarding, e opcao de skip.

## Notas Tecnicas
- **Visuais Dark Academia:** Usar icones SVG inline ou componentes React com estilo tematico (livros, penas, constelacoes, colunas gregas, ampulhetas). Evitar imagens raster para manter performance. Cores dos tokens HSL do design system (foreground, muted, accent). Considerar usar Lucide icons (ja no projeto via shadcn/ui) como base e estilizar.
- **Onboarding modal:** Usar `Dialog` do shadcn/ui como base. Steps gerenciados via state local (useState). Transicoes entre steps com CSS transitions ou framer-motion se ja estiver no projeto. Manter o modal acessivel (focus trap, ESC para fechar, ARIA labels).
- **localStorage vs DB:** Para MVP, usar localStorage para persistir `onboarding_completed`. Vantagem: zero latencia, funciona offline, sem migration. Desvantagem: nao sincroniza entre dispositivos. Para futuro, considerar coluna `onboarding_completed` na tabela de perfil do usuario no Supabase.
- **Deteccao de primeiro login:** Checar `localStorage.getItem('onboarding_completed')` no componente pai (layout ou page). Se `null` ou `'false'`, renderizar o modal de onboarding. Usar `useEffect` para evitar hydration mismatch (localStorage nao existe no server).
- **Hydration safety:** O onboarding modal deve ser renderizado client-side only (`'use client'`) com check de `typeof window !== 'undefined'` ou `useEffect` mount guard para evitar mismatch entre server e client render.
- **Empty state pattern:** Criar componente reutilizavel `EmptyState` com props: `icon` (ReactNode), `title` (string), `description` (string), `action` (opcional — { label: string, href: string } ou onClick). Usar em todos os 4 cenarios para consistencia.
- **Chat empty state existente:** O `chat-empty-state.tsx` ja existe. Aprimora-lo mantendo a interface existente e adicionando visuais. Verificar se aceita props de mind-specific suggestions — se nao, adicionar.
- **Acessibilidade:** Onboarding modal precisa de: `role="dialog"`, `aria-modal="true"`, `aria-label`, focus trap (shadcn Dialog ja fornece). Empty states: usar `role="status"` ou semantica adequada para screen readers. CTAs devem ter labels descritivos.
- **Responsividade:** Empty states em mobile devem reduzir tamanho dos visuais e usar stack vertical. Onboarding modal em mobile deve ser full-screen ou quase (max-w ajustado). Considerar `max-w-lg` desktop, `w-full mx-4` mobile.
- **Testes:** Para testes de localStorage, mockar `window.localStorage` com `vi.spyOn(Storage.prototype, 'getItem')` e `vi.spyOn(Storage.prototype, 'setItem')`. Para testes de Dialog, usar `@testing-library/react` com `screen.getByRole('dialog')`.
