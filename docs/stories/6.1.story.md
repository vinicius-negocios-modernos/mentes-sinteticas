# Story 6.1: Session Themes ‚Äî Temas Visuais por Mente
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** InProgress
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario do Mentes Sinteticas, quero que cada mente (Aristoteles, Da Vinci, Tesla, Curie, Hypatia, Turing) tenha um tema visual proprio aplicado a sessao de chat ‚Äî com cores de destaque, tons e elementos decorativos sutis que reflitam a personalidade e a era de cada mente ‚Äî para que a experiencia de conversa seja imersiva e visualmente distinta, refor√ßando a identidade unica de cada consciencia digital.

## Contexto
O app atualmente usa um tema dark unico (Deep Indigo base, Purple primary, Cyan accent) definido em `globals.css` via HSL custom properties. O sistema de design (shadcn/ui) ja consome essas variaveis, e o `next-themes` ja esta integrado para dark/light toggle. A base esta pronta para extensao por mente.

**O que ja existe (dependencias resolvidas):**
- Design System com HSL tokens em `src/app/globals.css` (`:root` e `.dark` scopes)
- shadcn/ui (New York) consumindo variaveis CSS via `@theme inline`
- `next-themes` integrado para dark mode
- DB schema `minds` com campos: `slug`, `name`, `title`, `era`, `personalityTraits` (`src/db/schema/minds.ts`)
- Chat page server component: `src/app/chat/[mindId]/page.tsx` ‚Äî ja resolve `mindData` (slug, nome, etc.)
- Chat components: `chat-interface`, `chat-message`, `chat-input`, `chat-header`, `chat-empty-state`
- Glass panel, text-gradient, e dark body gradient ja definidos em `globals.css`
- i18n infrastructure: `src/lib/i18n/` (t() function, pt-BR messages)
- 6 minds ativas no banco: aristoteles, da-vinci, tesla, curie, hypatia, turing

**Abordagem tecnica:**
O tema por mente sera implementado via CSS custom properties scoped. Um data attribute (`data-mind-theme="slug"`) aplicado no container do chat page altera as variaveis de accent/primary/decoracao, mantendo a base dark intacta. Isso evita conflito com o dark/light toggle do `next-themes` e permite extensao facil para novas mentes no futuro.

## Notas Tecnicas
- **CSS custom properties scoping:** A abordagem `[data-mind-theme="slug"]` e mais performatica que classes dinamicas porque o browser resolve as variaveis uma vez no cascade. Colocar o `data-mind-theme` no container mais externo do chat page garante que todos os filhos herdam as variaveis sem prop drilling.
- **Estrutura do seletor CSS:**
  ```css
  .dark [data-mind-theme="aristoteles"] {
    --primary: 38 92% 50%;     /* Amber/Gold */
    --accent: 45 93% 47%;      /* Golden */
    --ring: 38 92% 50%;
    --mind-gradient-from: #92400e;
    --mind-gradient-to: #1e1b4b;
    --mind-glow: rgba(245, 158, 11, 0.15);
  }
  ```
- **Fallback strategy:** O CSS usa as variaveis existentes como fallback natural ‚Äî se `data-mind-theme` nao esta presente, as variaveis do `.dark` scope prevalecem. O hook `useMindTheme` retorna um `defaultTheme` que espelha os valores atuais (purple/cyan).
- **Transicoes:** Aplicar `transition: --primary 400ms ease, --accent 400ms ease, ...` no container do chat. Nota: CSS custom properties NAO sao animaveis nativamente com `transition`. A alternativa e usar `@property` para registrar as variaveis como `<color>` (suporte: Chrome 85+, Safari 15.4+, Firefox 128+). Se o suporte for insuficiente, usar transicao de `opacity` no container com um brief fade (opacity 0 -> 1 em 300ms) ao montar o componente, simulando a transicao de tema.
- **Performance:** Evitar JavaScript runtime para aplicar cores ‚Äî tudo via CSS. O hook `useMindTheme` e para casos onde um componente precisa do valor programaticamente (ex: canvas, SVG dynamic). Os componentes React normais devem usar as classes Tailwind que referenciam as variaveis CSS (ex: `text-primary`, `bg-accent`).
- **Extensibilidade:** Para adicionar tema a uma nova mente no futuro: (1) adicionar entrada em `mind-themes.ts`, (2) adicionar seletor CSS em `globals.css`. Dois arquivos, zero mudanca em componentes.
- **Light mode:** Se no futuro light mode for habilitado, cada mente precisara de variantes light. Nesta story, focar apenas em `.dark` scope, que e o tema padrao.
- **Glow nas mensagens:** Usar `box-shadow: 0 0 20px var(--mind-glow)` no container da mensagem da mente. O valor de `--mind-glow` e rgba com alpha baixo (0.1-0.2) para ser sutil e nao distrair da leitura.
- **SSR safety:** O `data-mind-theme` e aplicado no server component (page.tsx), entao o CSS ja e resolvido no primeiro render ‚Äî sem flash de tema errado (FOUC). O hook `useMindTheme` e client-side mas so e usado para valores programaticos, nao para estilizacao principal.
