# Story 6.4: Rich Message Formatting — Code, LaTeX, Collapsible
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** Draft
**Estimativa:** 8 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario conversando com as Mentes Sinteticas, quero que as respostas da IA sejam renderizadas com formatacao rica — blocos de codigo com syntax highlighting, formulas matematicas/LaTeX, secoes colapsaveis para textos longos, e markdown completo (negrito, italico, listas, headers, tabelas, links) — para que o conteudo tecnico e cientifico discutido por mentes como Turing, Tesla, Curie e Hypatia seja apresentado de forma legivel, profissional e acessivel.

## Contexto
As respostas da Gemini API ja retornam conteudo em formato markdown. O componente `src/components/chat/chat-message.tsx` ja utiliza `react-markdown` para renderizacao basica, mas sem suporte a:
- **Syntax highlighting** em blocos de codigo (aparecem sem cores/formatacao)
- **LaTeX/formulas matematicas** (renderizadas como texto cru)
- **Secoes colapsaveis** para respostas longas
- **Copy button** especifico por bloco de codigo (existe apenas copy da mensagem inteira)

**Impacto por Mente:**
- **Turing:** Discute algoritmos, pseudocodigo, codigo — precisa de syntax highlighting
- **Tesla:** Circuitos, formulas eletromagneticas — precisa de LaTeX + code
- **Curie:** Equacoes de fisica, formulas quimicas — precisa de LaTeX
- **Hypatia:** Teoremas matematicos, provas — precisa de LaTeX
- **Da Vinci:** Descricoes tecnicas longas — beneficia de colapsaveis
- **Todas:** Respostas com listas, tabelas, headers — markdown completo

**O que ja existe:**
- `react-markdown` instalado e em uso no `chat-message.tsx`
- Tailwind typography plugin (`prose prose-invert`) aplicado ao container de mensagens
- Copy button global na mensagem (copia texto inteiro)
- Design System shadcn/ui com componente Collapsible disponivel
- Streaming via Vercel AI SDK `useChat` hook (mensagens chegam incrementalmente)

**Riscos:**
- Performance durante streaming: parsing markdown incremental pode causar flicker/re-render excessivo
- Bundle size: bibliotecas de syntax highlighting e LaTeX podem ser pesadas — lazy loading necessario
- LaTeX edge cases: formulas inline `$...$` vs block `$$...$$` precisam de tratamento distinto

## Notas Tecnicas
- **react-markdown:** Ja em uso (v9+). Suporta `remarkPlugins` e `rehypePlugins` como props. O prop `components` permite override de qualquer elemento HTML renderizado (ex: `code`, `pre`, `a`, `table`).
- **remark-gfm:** Plugin para GitHub Flavored Markdown. Adiciona suporte a tabelas, strikethrough (`~~text~~`), task lists (`- [x]`), e autolinks. Essencial para respostas ricas da IA.
- **remark-math + rehype-katex:** Pipeline em 2 etapas — `remark-math` faz parse dos delimitadores `$`/`$$` na AST markdown, `rehype-katex` renderiza os nodes math em HTML com KaTeX. Ambos necessarios.
- **rehype-highlight:** Usa highlight.js internamente. Detecta linguagem automaticamente quando nao especificada. Temas disponiveis como CSS — importar apenas o tema dark escolhido. Alternativa: `rehype-prism-plus` (usa Prism.js, mais leve mas menos linguagens).
- **KaTeX vs MathJax:** KaTeX escolhido por ser significativamente mais rapido no rendering (importante para streaming) e menor em bundle size (~100KB vs ~500KB para MathJax). Trade-off: KaTeX suporta menos comandos LaTeX que MathJax, mas cobre 95%+ dos casos de uso comuns.
- **Lazy loading strategy:** Usar `next/dynamic` com `ssr: false` para componentes pesados (KaTeX renderer). Highlight.js themes carregados como CSS dinamico. Alternativa: code splitting via dynamic `import()` dentro de `useMemo`.
- **Streaming + markdown:** `react-markdown` re-renderiza a cada update do texto. Para evitar flicker, memoizar o componente de mensagem e usar key estavel. Markdown incompleto (ex: ``` sem fechar) e tratado gracefully pelo parser — renderiza como texto ate completar.
- **Collapsible threshold:** 15 linhas escolhido como default baseado em UX — corresponde a ~1 tela em mobile. Considerar tornar configuravel via constante em `src/lib/constants.ts`.
- **Copy individual vs global:** O botao de copy global da mensagem (AC existente) permanece. O novo botao de copy por code block e ADICIONAL. Ambos coexistem.
- **Tailwind typography:** `prose prose-invert` ja aplicado. Os novos elementos (tabelas, code blocks) herdam esses estilos. Customizacoes adicionais via `prose-*` modifiers ou override no `components` prop.
- **CSS imports:** O CSS do highlight.js theme e do KaTeX devem ser importados. Opcoes: (1) import global no `layout.tsx`, (2) import no componente com lazy loading, (3) `<link>` tag condicional. Opcao 2 preferida para otimizar bundle.
- **Pacotes a instalar:** `remark-gfm`, `remark-math`, `rehype-highlight`, `highlight.js`, `rehype-katex`, `katex`, `@types/katex` (se necessario para TS).
