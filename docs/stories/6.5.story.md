# Story 6.5: Mind Profile Pages -- /mind/{slug}
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** InProgress
**Estimativa:** 4 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como visitante ou usuario autenticado do Mentes Sinteticas, quero que cada mente tenha uma pagina de perfil dedicada em `/mind/{slug}` (ex: `/mind/aristoteles`) -- para que eu possa conhecer melhor a mente antes de iniciar uma conversa, compartilhar o link do perfil em redes sociais com preview rico (OG/Twitter Cards), e para que buscadores indexem cada mente individualmente, aumentando a descoberta organica do app.

## Contexto
Atualmente, as mentes sao listadas apenas como tags/cards na home page, sem paginas individuais. O usuario so pode interagir com uma mente ao iniciar diretamente uma conversa em `/chat/{mindId}`. Isso limita:

1. **Descoberta:** Nao ha como compartilhar um link direto para uma mente especifica.
2. **SEO:** Buscadores nao conseguem indexar informacoes individuais de cada mente.
3. **Engajamento:** O usuario nao tem como explorar detalhes da mente (biografia, areas de expertise, fontes de conhecimento) antes de se comprometer com uma conversa.
4. **Compartilhamento social:** Links compartilhados sempre levam a home, sem contexto da mente especifica.

**O que ja existe (dependencias resolvidas):**
- DB schema `minds` com campos: `slug`, `name`, `title`, `era`, `nationality`, `systemPrompt`, `personalityTraits`, `greeting`, `avatarUrl`, `isActive` (`src/db/schema/minds.ts`)
- DB schema `knowledge_documents` com `displayName`, `description` por mente (`src/db/schema/knowledge-documents.ts`)
- Service `getMindBySlug(slug)` ja implementado (`src/lib/services/minds.ts`)
- Service `getMindWithDocuments(mindId)` ja implementado (`src/lib/services/minds.ts`)
- Design System shadcn/ui com glass-panel, tema dark academia (`src/components/ui/`)
- SEO infra: `opengraph-image.tsx`, `manifest.ts`, `robots.ts`, `sitemap.ts` (Story 5.1)
- Components: `MindCard`, `MindTag` (`src/components/minds/`)
- A11y baseline: semantic HTML, ARIA, focus-visible (Story 4.6)
- i18n infra: `src/lib/i18n/` com `t()` function (Story 5.5)

**Escopo desta story:**
Criar route `/mind/[slug]` com pagina de perfil completa, componentes de UI dedicados, SEO individual por mente (OG image dinamico, meta tags, structured data Person), e integracao com sitemap.

## Notas Tecnicas
- **Route structure:** `src/app/mind/[slug]/page.tsx` usa App Router dynamic segments. O parametro `slug` corresponde ao campo `minds.slug` no DB (ex: "aristoteles", "da-vinci", "tesla").
- **Data fetching:** `getMindBySlug` e `getMindWithDocuments` ja existem em `src/lib/services/minds.ts`. Nao criar servicos novos -- reutilizar os existentes.
- **SSG com `generateStaticParams`:** Buscar todos os slugs ativos via `listActiveMinds()` e retornar como array de params. Isso pre-renderiza todas as paginas de perfil no build.
- **`generateMetadata`:** Retornar `title: mind.name + " | Mentes Sinteticas"`, `description` derivada do title/personalityTraits, `openGraph` com imagem dinamica.
- **OG Image dinamico:** Usar `ImageResponse` do `next/og` (mesmo pattern de `src/app/opengraph-image.tsx`). Renderizar nome, titulo, e logo em canvas 1200x630 com fundo escuro.
- **Conversation starters:** Podem ser um campo `suggestedPrompts` no DB (futuro) ou hardcoded em um map `slug -> prompts[]` em `src/data/conversation-starters.ts`. Para esta story, hardcoded e suficiente -- migracao para DB pode ser story futura.
- **Prompt na URL:** Ao clicar em um starter, redirecionar para `/chat/{slug}?prompt={encodeURIComponent(text)}`. O chat page pode ler esse query param e pre-popular o input (melhoria opcional nesta story, obrigatoria apenas o link).
- **Avatar placeholder:** Se `avatarUrl` for null, renderizar um circulo com as iniciais da mente em estilo dark academia (fundo gradient, texto serif). Componente reutilizavel `MindAvatar`.
- **glass-panel:** Classe CSS ja definida no design system. Usar para cards e secoes do perfil.
- **Responsive breakpoints:** Mobile-first. Stack vertical ate `sm:`, layout horizontal/grid a partir de `md:`. CTA sticky bottom em mobile usa `fixed bottom-0` com safe-area padding.
- **i18n:** Usar `t()` de `src/lib/i18n/` para labels estaticos (ex: "Sobre", "Areas de Expertise", "Fontes de Conhecimento", "Iniciar Conversa"). Manter conteudo das mentes em portugues (dados do DB).
- **A11y:** Heading hierarchy (h1 para nome, h2 para secoes), landmarks (`<main>`, `<nav>`, `<section>`), alt text em avatar, `aria-label` no CTA, focus-visible em todos interativos.
- **Structured data Person:** Schema.org `Person` e apropriado para figuras historicas. Campos: `@type`, `name`, `jobTitle`, `description`, `url`, `sameAs` (links externos se disponivel no futuro).
- **Sitemap update:** Adicionar query `listActiveMinds()` no `src/app/sitemap.ts` e gerar entradas `/mind/{slug}` para cada mente. Manter as URLs existentes (/, /login, /signup).
- **Loading state:** `src/app/mind/[slug]/loading.tsx` com skeleton seguindo pattern existente em `src/components/skeletons/`.
