# Story 6.6: Conversation Sharing -- Link Publico
**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** Draft
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao
Como usuario do Mentes Sinteticas, quero poder compartilhar uma conversa com uma mente via link publico -- para que outras pessoas possam ler a conversa sem precisar de conta, ampliando a descoberta do app e permitindo que eu mostre interacoes interessantes com as mentes sinteticas.

## Contexto
Atualmente, todas as conversas sao privadas e acessiveis apenas pelo usuario autenticado que as criou. RLS policies restringem leitura/escrita ao `user_id` dono. Nao existe mecanismo para tornar uma conversa visivel publicamente. Isso limita:

1. **Viralidade:** Usuarios nao podem compartilhar conversas interessantes em redes sociais ou mensageiros.
2. **Descoberta:** Novos usuarios nao conseguem ver exemplos reais de conversas antes de criar conta.
3. **Marketing organico:** Cada conversa compartilhada e uma vitrine do produto.

**O que ja existe (dependencias resolvidas):**
- DB schema `conversations` com campos: `id`, `user_id`, `mind_id`, `title`, `created_at`, `updated_at` (`src/db/schema/conversations.ts`)
- DB schema `messages` com campos: `id`, `conversation_id`, `role`, `content`, `created_at` (`src/db/schema/messages.ts`)
- RLS policies user-scoped em conversations e messages
- Services: `src/lib/services/conversations.ts`, `src/lib/services/messages.ts`
- Chat components: `src/components/chat/` (chat-interface, chat-message, chat-input, chat-header, chat-empty-state, conversation-list)
- Auth: Supabase Auth com middleware protegendo `/chat/*`
- SEO infra: `opengraph-image.tsx`, `manifest.ts`, `sitemap.ts` (Story 5.1)
- Design System: shadcn/ui, dark academia theme, glass-panel
- A11y baseline: semantic HTML, ARIA, focus-visible (Story 4.6)
- i18n infra: `src/lib/i18n/` com `t()` function (Story 5.5)
- Haptics: `src/lib/haptics.ts` (Story 5.5)

**Escopo desta story:**
Adicionar mecanismo de sharing por token na tabela `conversations`, criar rota publica `/shared/[token]` para leitura read-only, UI de share/unshare no chat-header, e OG tags dinamicos para links compartilhados.

## Notas Tecnicas
- **Token generation:** Usar `crypto.randomBytes(32).toString('hex')` (Node.js built-in). Resultado: 64 caracteres hexadecimais. Entropia suficiente (256 bits) para ser impraticavel de adivinhar por forca bruta.
- **RLS approach:** A abordagem mais limpa e criar uma funcao RPC `get_shared_conversation(p_token text)` com `SECURITY DEFINER` que busca a conversa e suas mensagens quando `share_token` corresponde. Isso evita complexidade de policies com `current_setting`. A funcao retorna JSON com conversa + mensagens + mind info. Alternativa: service-level bypass com Supabase admin client (service_role key) na rota `/shared/[token]`.
- **Middleware exclusion:** O middleware de auth em `src/middleware.ts` ja protege `/chat/*`. A rota `/shared/*` nao precisa de exclusao explicita se o matcher nao inclui `/shared`. Verificar e ajustar o matcher se necessario.
- **Reuso de ChatMessage:** O componente `ChatMessage` de `src/components/chat/chat-message.tsx` deve ser reutilizado na shared view. Se ele depende de contexto de auth, extrair a renderizacao pura para um componente base `MessageBubble` e reutilizar em ambos.
- **Share URL format:** `{NEXT_PUBLIC_APP_URL}/shared/{token}`. Usar a env var `NEXT_PUBLIC_APP_URL` (ja existente da Story 4.8) para construir a URL completa.
- **Clipboard API:** Usar `navigator.clipboard.writeText(url)` com fallback para `document.execCommand('copy')` em browsers antigos. Tratar erro com toast de fallback ("Copie o link: {url}").
- **Haptic feedback:** Usar `triggerHaptic('confirm')` ao copiar link e `triggerHaptic('medium')` ao revogar. Importar de `src/lib/haptics.ts`.
- **i18n:** Usar `t()` de `src/lib/i18n/` para todos os labels: "Compartilhar", "Link copiado!", "Revogar compartilhamento", "Tem certeza?", "Crie sua conta para conversar", etc.
- **OG image:** Seguir pattern de `src/app/opengraph-image.tsx` (Story 5.1). Renderizar nome da mente, titulo da conversa, e branding em canvas 1200x630 com fundo escuro estilo dark academia.
- **Rate limiting:** Reutilizar infraestrutura de `src/lib/services/rate-limiter.ts` (Story 2.6). Adicionar novas regras: `share:create` (10/hora por user) e `shared:view` (60/minuto por IP).
- **Revoke confirmation:** Usar `AlertDialog` do shadcn/ui para confirmacao antes de revogar. Texto: "Ao revogar, o link atual deixara de funcionar. Deseja continuar?"
- **CTA na shared view:** Link "Crie sua conta para conversar" aponta para `/signup`. Botao secundario "Conhecer mais mentes" aponta para `/`.
- **A11y na shared view:** Heading hierarchy (h1 titulo, h2 secoes), `<main>` landmark, `role="log"` na lista de mensagens (mesmo pattern do chat), `aria-label` nos CTAs.
- **Loading state:** `src/app/shared/[token]/loading.tsx` com skeleton reutilizando pattern de `src/components/skeletons/chat-messages-skeleton.tsx`.
- **Cascade on delete:** Quando uma conversa e deletada, o `share_token` e invalidado automaticamente. A rota publica `/shared/{token}` deve retornar 404. Implementar via `ON DELETE CASCADE` ou verificacao no handler.
