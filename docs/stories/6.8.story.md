# Story 6.8: Ambient Soundscapes

**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** Draft
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao

Como usuario do Mentes Sinteticas, quero que cada sessao de chat tenha uma ambientacao sonora sutil correspondente a era e personalidade da mente, para que a experiencia seja mais imersiva e atmosferica.

Adicionar paisagens sonoras ambientes (ambient soundscapes) as sessoes de chat, criando uma atmosfera auditiva imersiva que complementa o tema visual de cada mente. Cada uma das 6 mentes tera um loop de audio ambiente unico, inspirado na sua era e personalidade. O usuario tera controle total sobre volume e mute, com crossfade suave ao trocar de mente e respeito a preferencias de acessibilidade.

## Notas Tecnicas

- **Web Audio API vs HTMLAudioElement:** Preferir Web Audio API (`AudioContext`, `GainNode`) para controle preciso de volume e crossfade. `HTMLAudioElement` pode ser usado como source node.
- **Crossfade:** Dois `GainNode` — um fazendo ramp down, outro ramp up, ambos conectados ao destination. Usar `linearRampToValueAtTime` para transicoes suaves.
- **Loop sem gap:** Usar `AudioBufferSourceNode.loop = true` ou detectar fim do `HTMLAudioElement` e reiniciar. Testar que nao ha click/pop na transicao.
- **Mobile autoplay:** Safari e Chrome mobile bloqueiam `AudioContext` ate user gesture. Criar AudioContext no primeiro `click`/`touchstart` global via `AudioContext.resume()`.
- **Bundle impact:** O modulo JS de audio e leve (~2-3KB). Os arquivos `.webm` sao servidos sob demanda (nao entram no bundle JS).
- **Padrao haptics:** `src/lib/haptics.ts` ja implementa deteccao de `prefers-reduced-motion` — reutilizar a mesma abordagem (media query match).
- **SSR safety:** `AudioContext` e `window.matchMedia` so existem no browser. Toda inicializacao deve ser lazy (dentro de `useEffect` ou event handler). Nunca importar modulo de audio no server.
