# Story 6.9: Accessibility Pass 2 — Full WCAG AA

**Epic:** 1 -- Transformacao Mentes Sinteticas
**Fase:** 6 -- Features Legendarias
**Status:** Draft
**Estimativa:** 5 SP (1 SP = ~2h)
**Criado por:** @sm (River)

## Descricao

Como usuario do Mentes Sinteticas com necessidades de acessibilidade, quero que todas as features da Fase 6 atendam ao padrao WCAG 2.1 AA, para que eu possa usar o produto com tecnologias assistivas de forma completa e sem barreiras.

Auditoria abrangente de acessibilidade WCAG 2.1 AA apos todas as features da Fase 6 estarem implementadas. Este e o passo final de acessibilidade antes do produto ser considerado totalmente acessivel. Combina testes automatizados (axe-core, Lighthouse), testes manuais com screen readers (VoiceOver), auditoria de navegacao por teclado, e verificacao de suporte a reduced motion para todas as animacoes introduzidas na Fase 6.

## Contexto

A Story 4.6 (A11y Pass 1) estabeleceu a fundacao de acessibilidade:
- Skip-to-content link
- HTML semantico e hierarquia de headings
- ARIA live regions no chat (role="log")
- Labels em formularios (login/signup)
- Alt text em imagens
- Contraste de cores WCAG AA (text-muted-foreground)
- Focus-visible global
- role="article" nos mind cards

A Fase 6 introduziu features complexas que requerem revisao de acessibilidade dedicada:
- **6.1 Session Themes:** Novas paletas de cores que precisam de verificacao de contraste
- **6.2 Multi-Mind Debates:** Multiplas mentes falando — screen readers precisam identificar quem esta falando
- **6.3 Mind Memory:** Configuracoes persistentes — formularios e controles acessiveis
- **6.4 Rich Formatting:** Markdown/code blocks — estrutura semantica correta
- **6.5 Mind Profiles:** Novas paginas/modais — focus management
- **6.6 Conversation Sharing:** Links e modais de compartilhamento — anuncios de estado
- **6.7 Voice Mode:** Controles de audio acessiveis, estados anunciados (gravando, processando, reproduzindo)
- **6.8 Ambient Soundscapes:** Audio automatico respeitando preferencias, controles mutaveis

## Criterios de Aceite

### AC1: Auditoria Automatizada — Zero Violacoes Criticas
- [ ] axe-core integrado nos testes (via @axe-core/react ou axe-playwright)
- [ ] Lighthouse Accessibility score >= 95 em todas as paginas principais (/, /chat/[mindId], /login, /signup, /mind/[id])
- [ ] Zero violacoes de severidade "critical" ou "serious" no axe-core
- [ ] Violacoes "moderate" documentadas com justificativa se nao resolvidas

### AC2: Contraste de Cores — Todos os Temas
- [ ] Cada Session Theme (6.1) passa WCAG AA contrast ratio (4.5:1 texto normal, 3:1 texto grande)
- [ ] Cores de identificacao de mentes nos debates (6.2) tem contraste suficiente contra backgrounds de todos os temas
- [ ] Estados interativos (hover, focus, active, disabled) manteem contraste em todos os temas
- [ ] Indicadores de estado do Voice Mode (6.7) visiveis em todos os temas

### AC3: Navegacao por Teclado — Features Novas
- [ ] Debate view (6.2): Tab navega entre mensagens de diferentes mentes; Enter/Space ativa controles de debate
- [ ] Mind Memory settings (6.3): Todos os controles (toggles, inputs, buttons) acessiveis por teclado
- [ ] Rich formatting toolbar (6.4): Toolbar navegavel por arrow keys, Escape fecha menus
- [ ] Mind Profile pages (6.5): Navegacao completa por Tab, acoes por Enter/Space
- [ ] Share dialog (6.6): Focus trap no modal, Escape fecha, Tab cicla entre opcoes
- [ ] Voice Mode controls (6.7): Spacebar para gravar, Escape para cancelar, Tab entre controles
- [ ] Soundscape controls (6.8): Controles de volume e mute acessiveis por teclado

### AC4: Focus Management — Modais e Conteudo Dinamico
- [ ] Focus trap em todos os modais e drawers novos (profiles, sharing, settings)
- [ ] Focus retorna ao elemento trigger quando modal/drawer fecha
- [ ] Novo conteudo dinamico (mensagens de debate, respostas de voz) nao rouba focus do usuario
- [ ] Skip links atualizados para incluir novas landmarks (debate area, voice controls)

### AC5: Screen Reader — Anuncios e Estrutura
- [ ] VoiceOver (macOS): Navegacao completa por todas as features da Fase 6 sem bloqueios
- [ ] Debates (6.2): Screen reader anuncia qual mente esta falando antes de cada mensagem (ex: "Aristoteles disse:")
- [ ] Voice Mode (6.7): Estados anunciados via aria-live — "Gravando", "Processando", "Reproduzindo resposta"
- [ ] Soundscapes (6.8): Estado anunciado — "Som ambiente ativado: Biblioteca", "Som ambiente desativado"
- [ ] Rich content (6.4): Code blocks anunciados como "bloco de codigo", listas como listas semanticas
- [ ] Conversation sharing (6.6): Confirmacao anunciada — "Link copiado para area de transferencia"

### AC6: Reduced Motion — Animacoes da Fase 6
- [ ] `prefers-reduced-motion: reduce` desabilita todas as animacoes decorativas da Fase 6
- [ ] Transicoes de tema (6.1) respeitam preferencia — troca instantanea sem fade
- [ ] Animacoes de debate (6.2) — indicadores de "digitando" substituidos por texto estatico
- [ ] Voice Mode (6.7) — animacao de waveform substituida por indicador estatico
- [ ] Soundscape visualizations (6.8) — desabilitadas, apenas controles estaticos

### AC7: Voice Mode — Acessibilidade Especifica
- [ ] Botao de gravar tem aria-label descritivo ("Iniciar gravacao de voz" / "Parar gravacao")
- [ ] Estado do botao refletido via aria-pressed ou aria-expanded
- [ ] Transcricao visivel durante gravacao (feedback visual alem do audio)
- [ ] Controles de playback (pausar, parar) acessiveis e anunciados
- [ ] Fallback textual sempre disponivel — voice mode nunca e o unico meio de input

### AC8: Soundscapes — Acessibilidade de Audio
- [ ] Soundscapes respeitam `prefers-reduced-motion` (nao iniciam automaticamente)
- [ ] Controle de mute sempre visivel e acessivel por teclado
- [ ] Volume ajustavel por teclado (arrow keys no slider)
- [ ] aria-label no controle de volume inclui valor atual ("Volume: 70%")
- [ ] Soundscapes nao interferem com screen readers (pausam durante anuncios ou usam canal separado)

### AC9: Formularios e Controles Novos
- [ ] Mind Memory settings (6.3): Labels associados, error messages com aria-describedby
- [ ] Share form (6.6): Labels em todos os inputs, estados de erro anunciados
- [ ] Todos os novos toggles/switches tem role="switch" e aria-checked
- [ ] Grupos de radio/checkbox agrupados com fieldset/legend ou role="group"

### AC10: Landmarks e Estrutura de Pagina
- [ ] Hierarquia de headings (h1-h6) correta em todas as novas paginas/views
- [ ] Novas sections tem aria-label ou aria-labelledby descritivo
- [ ] Debate view usa landmarks semanticos (main, complementary para sidebar de mentes)
- [ ] Mind Profile pages seguem estrutura: h1 (nome), h2 (secoes), conteudo

### AC11: Documentacao de Acessibilidade
- [ ] Arquivo `docs/accessibility.md` criado com:
  - Padroes de acessibilidade seguidos (WCAG 2.1 AA)
  - Keyboard shortcuts documentados
  - Screen readers testados e suportados
  - Limitacoes conhecidas (se houver)
  - Como reportar issues de acessibilidade

## Notas Tecnicas

### Ferramentas de Auditoria
- **axe-core:** `@axe-core/playwright` para testes E2E, `vitest-axe` para testes unitarios
- **Lighthouse:** Via Chrome DevTools ou `lighthouse` CLI
- **VoiceOver:** Nativo macOS (Cmd+F5), testar Safari e Chrome
- **Contrast checker:** Usar ferramentas como WebAIM Contrast Checker para temas novos

### Padroes ARIA para Debates (6.2)
```html
<div role="log" aria-label="Debate entre mentes">
  <div role="article" aria-label="Aristoteles disse">
    <span class="sr-only">Aristoteles:</span>
    <p>Conteudo da mensagem...</p>
  </div>
</div>
```

### Padroes ARIA para Voice Mode (6.7)
```html
<button
  aria-label="Iniciar gravacao de voz"
  aria-pressed="false"
>
  <MicIcon aria-hidden="true" />
</button>
<div aria-live="polite" aria-atomic="true">
  Gravando...
</div>
```

### Padroes ARIA para Soundscapes (6.8)
```html
<div role="group" aria-label="Controles de som ambiente">
  <button aria-label="Silenciar som ambiente" aria-pressed="false">
    <VolumeIcon aria-hidden="true" />
  </button>
  <input
    type="range"
    role="slider"
    aria-label="Volume do som ambiente"
    aria-valuemin="0"
    aria-valuemax="100"
    aria-valuenow="70"
    aria-valuetext="70%"
  />
</div>
```

### Referencia WCAG 2.1 AA — Criterios Chave
- **1.1.1** Non-text Content (alt text)
- **1.3.1** Info and Relationships (semantic HTML)
- **1.4.3** Contrast Minimum (4.5:1)
- **1.4.11** Non-text Contrast (3:1 UI components)
- **2.1.1** Keyboard (all functionality)
- **2.4.3** Focus Order (logical)
- **2.4.7** Focus Visible
- **2.5.5** Target Size (44px — ja implementado)
- **3.3.2** Labels or Instructions
- **4.1.2** Name, Role, Value (ARIA)
- **4.1.3** Status Messages (aria-live)
